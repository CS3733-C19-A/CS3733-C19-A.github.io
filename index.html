Alcohol and Tobacco Tax and Trade Bureau

COLA System User Manual

Max Westwater, Jonathan Dang, Shuxing Li, Christopher Bell, Jonathan Tai, James Witt, Dillon Arnold, Junyu Lyu, Alex Rus, Zhihao Xie

Overview

The TTB COLA search application was designed so that manufacturers and agents can better submit and approve alcohol label applications. This way, not only will the productivity of agents be increased but the manufacturers will also be able to easily submit applications for their new projects. The search function has also been improved so that it is much easier to research or look up approved TTB applications.

Click on the section headers to see a view of the section within the application.

Home Page

The Home Page contains two majors elements, and serves as the launching point from which users will initially interact with the TTB COLA search application.

  • COLA Search
    • This button takes you to the COLA Search page
  • Login
    • This button takes you to the Login Page
  • About
    • This button takes you to the About Page
  • Dark Mode
    • This toggle switch turns on and off Dark Mode, which changes the stylesheet of the application to be more suitable to night time usage

Navigation Bar

The Navigation Bar is the main way to get around the website. By clicking on the logo, you are taken back to the Home Page. When a user is not logged in, the two buttons at the top of the bar are Login and Sign Up which take you to the Login and Create Account pages respectively. When the user has logged in, then the buttons change to :

  • COLA Search
    • This button takes you to the COLA Search page
  • User Portal
    • This button takes the user to their User Portal
  • Logout
    • This button allows the user to logout of their account
If the user is an Agent they will see an extra button, User Statistics, which takes the user to the User Statistics Page.

COLA Search

The search function is the primary method for a user to interact with the TBB Database. Through the COLA Search Page, users are able to search for alcohol and view their application information.

  • Search Bar
    • The search bar allows you to type in keywords for alcohol that you might be looking for. It is able to suggest results that you might be looking for. Pressing the Search button will initiate the search.
    • Next to the search button you are able to choose the type of algorithm you would like to search with, depending on the kinds of results you are looking for
    • The right half of the search bar allows you to download the search results in a CSV file format with a chosen Delimiter. You can choose where on your computer you want to save your file to by clicking on the folder icon. You can also specify the desired Delimiter for separating the fields in the CSV by entering it into the Delimiter field. Once you are ready, clicking the Download Results button, downloads the CSV file.
  • Filters
    • The left bar allows a user to filter their search even further through filtering for the category of alcohol and where the alcohol was been sourced from
  • Search Results
    • Search results are displayed in the center of the page and are displayed in an infinite scrolling method where more results are loaded as you scroll towards the bottom of the page
    • Clicking on a search result creates an application pop up window which shows all the information of the selected alcohol and its application

Create Account

The Create Account Page serves as the page where individual manufacturers or agents can come to create new accounts, if they do not already have one.

  • The Agent and Manufacturer radio buttons allow an individual to choose what kind of account they are making, depending on who they are
  • The following fields are necessary to be filled out for each account creation:
    • Username
    • Password
    • Confirm Password
    • First Name
    • Last Name
    • Email
    • Phone Number
  • If the individual who is creating the account is an agent, they must select their specialty from the specialty dropdown
  • The two factor toggle switch toggles on or off the two-factor login security feature

Setup Two-Factor Authentication Page

Two-factor authentication utilizes Google Authenticator to initiate the two-factor security feature. By scanning the QR Code, your account will be linked to your phone. The user is able to enter a code that Google Authenticator produces in order to securely login to their account.

Login Page

The login page prompts the user to input their username and password and has three buttons:

  • Login
    • If the user's login credentials are correct, the user is logged in and taken to the User Portal
    • If the user's account has two-factor login enabled, the two-factor login page is prompted before they are able to log in to the User Portal.
  • No Account? Create one
    • If the user does not have an account, they are taken to the Account Creation Page
  • Reset Password
    • If the user has forgotten their password, they are taken to the Reset Password pop-up screen

Reset password

In this window, the user is prompted to enter their username. Once their username is entered, they press the Send Email button which sends recovery instructions to the user's email if the email exists. The user can then enter the verification code, as well as a new password, and click the reset button to reset their password.

User Portal

The User Portal serves as the main platform from which Agents and Manufacturers will be able to interact with the TTB Applications.

Agent

  • The left column displays the user information. For the Agent, this includes the agents specialty and the amount of applications that are currently unassigned
  • The main table displays the number of applications that are currently assigned to the agent. Clicking on any of the view buttons will take the Agent to the respective form approval page for the specific form
  • The Retrieve More button allows an Agent to populate their inbox with more forms review
  • The Update Account button takes the Agent to the Account Settings Page

Manufacturer

  • The Manufacturer User Portal has a column on the left that displays the basic account information
  • The main window has a table that displays all of the applications that have been submitted by the Manufacturer. Clicking any of the view buttons will allow the Manufacturer to view the submitted form
  • The Create New Application button allows the Manufacturer to create a new application to be submitted. The type of view that the submission page is set to can be toggled between Multi-Page or Single-Page scrolling through the Fill in Multi-Page toggle switch
  • The Update Account button takes the Manufacturer to the Account Settings Page

Account Settings

The Account Settings Page allows the user to change details about their account. Items that can be changed are as follows:

  • Password
  • Phone Number
  • Email
  • Two-Factor Authentication On/Off

Forms

TTB Application forms are used to approve and keep a registry of all alcoholic beverages ever sold and manufactured in the United States of America.

Submitting

  • The form submission for manufacturers is a digital replication of the physical form that manufacturers have to submit to TTB for new alcoholic products.
  • The form can appear in two forms, either multi-page or single-page, depending on the Manufacturer's selected preferences

Approving

  • An Agent can access the Form Approval Page in order to approve, reject, or forward applications
  • Approving an application automatically sends it back to the Manufacturer with the approval status
  • Rejecting the application allows the Agent to send the application back to the Manufacturer with feedback
  • Forwarding allows the Agent to forward the current application to a different Agentâ€™s inbox

Help Pages

The help buttons are buttons that appear near fillable text fields. In order to use the button you need to click or hover over the help button in order to show the pop-up window that displays the detailed help information.

About Page

The about page displays information about the software development team responsible for this project.

Other

Other special features that are added into the application by Team A are as follows:

GamificationSurprise!

  • A badge reward system is used to encourage higher productivity amongst agents. By completing more applications, the agents receive points which go to a point total which can be used to motivate the Agent
  • User Statistics is a page that only shows up for an Agent. It allows the Agent to see what kind of badges they have earned as well as the amount of points that they have

Voice Assistant

  • We have added Google Voice Assistant functionality, this allows the application to be accessible to those with visual impairments through voice commands

Email Notifications

  • If a password reset has been requested, or an application form has been rejected or accepted, an email notification will be sent to the email that is linked to the current account

OCR Image Search

  • A search feature has been implemented that is driven by uploading an image. The feature interprets the image and populates a text field for searching the database based on what it thinks the label name is.
